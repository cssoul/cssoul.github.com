以上方法的优缺点

方法一：HTTP请求数量多，影响网站性能；维护成本低。适合搭建TMS页面，运营可编辑、更改图片。
方法二：HTTP请求数量少；维护成本高。适合处理高访问量、不经常更新的页面。（比如：淘宝首页）
方法三：减少HTTP请求、利于扩展；维护成本较高，存在浏览器兼容问题。适用于Mobile Web开发。
方法四：减少HTTP请求；会增加页面大小，不能被浏览器缓存,同时也存在兼容问题。

还有一些浏览器的兼容问题...

IE： 只支持.eot格式，IE9开始支持.woff；
Firefox： 支持.ttf和.otf，Firefox 3.6+支持.woff；
Opera： 支持.ttf、.otf、.svg，Opera 11开始支持.woff；
Chrome： 支持.ttf、.otf，Chrome 6+支持woff格式；
safari： 支持.ttf、.otf格式字体

iOS、Android对字体格式支持情况

IOS Webkit：4.2+支持.ttf、5.1+开始支持.woff，4.2以下只支持SVG字体。
Android Webkit：2.2版本以上支持.ttf格式字体。
结论：在不需要兼容低配手机情况下，Mobile Web推荐使用.ttf格式字体。

如何运用icon字体到页面中？

先了解一下@font-face

@font-face是CSS3中的一个模块，它主要是把自己定义的Web字体嵌入到你的网页中。
Ⅰ、引进自定义字体：
                    	@font-face {
                            font-family: 'somefont'; 
                            src: url('somefont.ttf');
                            font-weight: 700;
                            font-style: normal;
                        }
Ⅱ、应用字体到元素上：
                        .icon{
                            font-family: 'somefont'; 
                            font-weight: 700;
                            font-style: normal;
                        }
